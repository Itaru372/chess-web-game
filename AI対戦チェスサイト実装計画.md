# AI対戦チェスサイト実装計画

本ドキュメントは、AIとの対戦が可能なチェスサイトの具体的な実装計画を概説します。ユーザーフレンドリーなインターフェース、リアルタイムのスコアリング、および永続的なレーティングシステムを特徴とするウェブアプリケーションの構築を目指します。

## 1. システムアーキテクチャ

このチェスサイトは、最新のウェブ技術を活用したクライアントサイドアプリケーションとして設計されます。フロントエンドは標準的なHTML、CSS、JavaScriptで構築され、チェスゲームのロジックとUIの描画には既存のJavaScriptライブラリを統合します。AIエンジンはWebWorkerを利用してブラウザ上で動作し、ユーザーインターフェースの応答性を維持します。

主要な技術スタックは以下の通りです。

| カテゴリ       | 技術要素           | 役割                                   |
| :------------- | :----------------- | :------------------------------------- |
| **フロントエンド** | HTML5, CSS3, JavaScript (ES6+) | ユーザーインターフェースの構築とインタラクション |
| **チェスロジック** | `chess.js`         | 指し手の検証、ゲーム状態の管理、合法手の生成 |
| **UIコンポーネント** | `chessboard.js`    | チェス盤の描画、駒のドラッグ＆ドロップ操作 |
| **AIエンジン**     | `Stockfish.js`     | ブラウザ上でのAI思考、対戦相手の提供       |

`chess.js`はチェスのルールを厳密に適用し、合法的な指し手のみを許可することでゲームの整合性を保ちます。`chessboard.js`は視覚的に魅力的なチェス盤を提供し、ユーザーが直感的に駒を操作できるようにします。`Stockfish.js`は、WebWorkerとしてバックグラウンドで動作させることで、AIが複雑な計算を行っている間もメインスレッドがブロックされず、スムーズなユーザー体験を実現します。

## 2. 対戦中のスコアシステム

対戦中には、各プレイヤーが獲得した駒の価値に基づいたリアルタイムのスコアが表示されます。これにより、ユーザーはゲームの有利・不利を視覚的に把握できます。駒の基本的な価値は以下の表に基づきます。

| 駒の種類 | 価値 (点) |
| :------- | :-------- |
| ポーン   | 1         |
| ナイト   | 3         |
| ビショップ | 3         |
| ルーク   | 5         |
| クイーン   | 9         |
| キング   | 無限      |

スコアは、**「自分が取った駒の合計点」から「相手に取られた駒の合計点」を差し引いた値**として計算されます。例えば、相手のポーンを1つ取るとスコアは+1となり、自分のナイトが取られるとスコアは-3となります。このスコアは盤面の横に「+X」または「-Y」の形式で表示され、現在の優劣を簡潔に示します。

## 3. ユーザーインターフェース (UI/UX)

ユーザーインターフェースは、初心者から経験者まで、誰にとっても分かりやすく直感的な操作性を提供することを目指します。

### ホーム画面

ホーム画面は、ユーザーが対戦を開始し、自身の成績を確認するための中心的なハブとなります。

- **「対局開始」ボタン**: 新しいAI対戦を開始します。
- **現在のレーティング**: ユーザーの現在のレーティング（初期値200）が表示されます。これは、対戦成績に基づいて変動する数値です。
- **過去の対戦成績**: 総対戦数、勝利数、敗北数、勝率などが一覧で表示されます。
- **直近の対戦履歴**: 過去数回の対戦結果（勝敗、相手AIレベル、レーティング変動など）がリスト形式で表示されます。

### 対局画面

対局画面は、チェスゲームの進行に特化したレイアウトとなります。

- **チェス盤**: `chessboard.js`によって描画される、視覚的にクリアで操作しやすいチェス盤です。
- **取られた駒のリスト**: 各プレイヤーが相手から取った駒が、盤面の横にアイコンで表示されます。
- **リアルタイムスコア**: 前述のスコアシステムに基づき、現在の有利・不利が数値で表示されます。
- **操作ボタン**: 「投了」（ゲームを諦める）や「リセット」（現在のゲームを最初からやり直す）などのボタンが配置されます。

## 4. レーティングシステム

ユーザーの対戦スキルを数値化するために、レーティングシステムを導入します。これは、対戦結果に基づいてユーザーのスキルレベルを評価するもので、初期値は**200**とします。レーティングはマイナスになることはなく、最低値は0です。

レーティングの変動ルールは以下の通りです。

- **勝利時**: レーティングは**+15〜25点**増加します。この増加量は、対戦相手のAIレベルや現在のレーティング差に応じて変動させることで、より公平な評価を目指します。
- **敗北時**: レーティングは**-10〜20点**減少します。同様に、相手のAIレベルやレーティング差に応じて変動させることが可能です。

このレーティングデータは、ブラウザの`localStorage`に保存されます。これにより、ユーザーがブラウザを閉じても、次回のアクセス時に前回のレーティングと対戦履歴が保持されます。

## 5. AI対戦の仕組み

AI対戦は、強力なオープンソースチェスエンジンである`Stockfish`のJavaScript版である`Stockfish.js`を利用して実現します。`Stockfish.js`はWebWorkerとして実装されるため、AIが思考に時間を要する場合でも、ユーザーインターフェースがフリーズすることなく、スムーズな操作が可能です。

- **難易度設定**: AIの強さは、レベル1から10までの範囲で設定できるようにします。これにより、ユーザーは自身のスキルレベルに合わせた挑戦が可能です。将来的には、ユーザーのレーティングに基づいてAIレベルを自動調整する機能も検討できます。
- **WebWorkerの活用**: AIの思考プロセスはメインスレッドとは別のWebWorkerで実行されます。これにより、AIが複雑な計算を行っている間も、ユーザーはチェス盤の操作や他のUI要素とのインタラクションを継続できます。
